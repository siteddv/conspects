# Полная инструкция по созданию конспекта книги

## Подготовка

1. **Помести PDF файл книги** в папку `conspects/[название_книги]-summary/original/`

2. **Найди страницы с оглавлением** в PDF (обычно в начале книги)

3. **Убедись, что есть скрипт для чтения PDF:**
   - Файл `read_pdf_pages.py` в корне проекта
   - Установлена библиотека: `pip install pypdf`

## Универсальный запрос

Скопируй и заполни следующий шаблон:

---

```
Мне нужно создать конспект книги по структуре, аналогичной @conspects/data-intensive-applications-summary.

**PDF файл:** conspects/[название_книги]-summary/original/[файл].pdf
**Страницы с оглавлением:** [X] - [Y]
**Корневая папка:** conspects/[название_книги]-summary/ru/

Создай конспект в три уровня детализации:
1. **полный/** - детальный конспект с подтемами
   - Структура: Часть → Глава → подтемы/
   - Каждая подтема в отдельном файле
   - Файл главы содержит ссылки на все подтемы
   - Файл части содержит ссылки на все главы

2. **по главам/** - конспект по главам без подтем
   - Объем в 3 раза меньше полного
   - Подзаголовки - ссылки на подтемы из полный/
   - В конце каждой главы - ссылка на полную главу

3. **краткий/** - единый конспект всей книги
   - Один файл: Конспект книги.md
   - 25000-30000 символов
   - Без явного упоминания глав и частей
   - Единое повествование

**Процесс создания:**

1. **Чтение оглавления:**
   - Прочитай страницы [X-Y] из PDF
   - Сохрани структуру в table_of_contents.txt
   - Определи иерархию: Части → Главы → Подтемы

2. **Создание полного конспекта (в цикле):**
   Для каждой подтемы:
   - Очисти контекст
   - Прочитай соответствующие страницы из PDF
   - Создай краткий пересказ (сохрани всю важную информацию, но сжато)
   - Сохрани: полный/[Часть X]/Глава Y/подтемы/[Y.Z. Название].md
   
   После всех подтем главы:
   - Прочитай все файлы подтем
   - Создай файл главы с ссылками на подтемы
   - Сохрани: полный/[Часть X]/Глава Y/Глава Y. [Название].md
   
   После всех глав части:
   - Прочитай все файлы глав
   - Создай файл части с ссылками на главы
   - Сохрани: полный/[Часть X]/Часть X. [Название].md

3. **Создание конспекта по главам:**
   Для каждой главы:
   - Прочитай все подтемы из полный/
   - Объедини в один конспект (в 3 раза короче)
   - Подзаголовки → ссылки на подтемы: ## [[Y.Z. Название]]
   - В конце → ссылка на полную главу
   - Сохрани: по главам/[Часть X]/главы/Глава Y. [Название].md
   
   После всех глав части:
   - Создай файл части с ссылками на главы
   - Сохрани: по главам/[Часть X]/Часть X. [Название].md

4. **Создание краткого конспекта:**
   - Прочитай все файлы глав из по главам/
   - Создай единое повествование (25000-30000 символов)
   - Без упоминания "Глава X" или "Часть Y"
   - Сохрани: краткий/Конспект книги.md

5. **Создание оглавлений:**
   - полный/Оглавление.md - все части, главы и подтемы
   - по главам/Оглавление.md - только части и главы

6. **Создание корневого файла:**
   - В корне: [Название книги].md
   - Содержимое:
     - Название книги
     - Автор
     - Оригинальное название
     - Краткое описание
     - Ссылки на три уровня конспектов

7. **Добавление markdown-ссылок:**
   - Во всех файлах рядом с [[...]] добавь [...](относительный_путь.md)
   - Относительные пути от текущего файла

**Требования:**
- Используй Obsidian-ссылки [[название]] или [[путь|текст]]
- Все файлы в формате Markdown (.md)
- Сохраняй всю важную информацию, но в сжатом виде
- Структура папок должна точно соответствовать структуре книги

Начни с чтения оглавления. Если есть вопросы, задай их перед началом.
```

---

## Структура папок (результат)

```
conspects/[название_книги]-summary/
├── [Название книги].md          # Корневой файл с описанием
├── original/
│   └── [файл].pdf              # PDF книги
└── ru/
    ├── полный/
    │   ├── Оглавление.md
    │   └── [Часть X]/
    │       ├── Часть X. [Название].md
    │       └── Глава Y/
    │           ├── Глава Y. [Название].md
    │           └── подтемы/
    │               └── [Y.Z. Название].md
    ├── по главам/
    │   ├── Оглавление.md
    │   └── [Часть X]/
    │       ├── Часть X. [Название].md
    │       └── главы/
    │           └── Глава Y. [Название].md
    └── краткий/
        └── Конспект книги.md
```

## Примеры ссылок

**В файле главы (по главам/):**
```markdown
## [[1.1. Подходы к работе]] [1.1. Подходы к работе](относительный_путь.md)

Текст конспекта...

**Подробнее:** [[Глава 1. Название]] [Глава 1. Название](относительный_путь.md)
```

**В оглавлении:**
```markdown
[[Часть 1. Название]] [Часть 1. Название](относительный_путь.md)

[[Глава 1. Название]] [Глава 1. Название](относительный_путь.md)
[[1.1. Подтема]] [1.1. Подтема](относительный_путь.md)
```

## Важные замечания

1. **Нумерация страниц:** В PDF нумерация обычно начинается с 1, но могут быть римские цифры для предисловия
2. **Структура книги:** Некоторые книги не имеют частей - только главы
3. **Подтемы:** Если в книге нет явных подтем, используй заголовки второго уровня (##) как подтемы
4. **Объем конспекта:** Адаптируй под размер книги (для коротких книг краткий конспект может быть меньше)
