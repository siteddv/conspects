# Компактный шаблон запроса для создания конспекта

## Быстрый запрос (скопируй и заполни):

```
Мне нужно создать конспект книги по структуре, аналогичной @conspects/data-intensive-applications-summary.

**PDF файл:** [путь_к_pdf_файлу]
**Страницы с оглавлением:** [номер_первой] - [номер_последней]
**Корневая папка:** conspects/[название_книги]-summary/ru/

Создай конспект в три уровня детализации:
1. **полный/** - детальный конспект с подтемами (структура: Часть → Глава → подтемы/)
2. **по главам/** - конспект по главам без подтем (объем в 3 раза меньше полного)
3. **краткий/** - единый конспект всей книги (25000-30000 символов, без упоминания глав/частей)

**Процесс:**
1. Прочитай оглавление из PDF (страницы [X-Y]), сохрани в table_of_contents.txt
2. В цикле для каждой подтемы:
   - Очисти контекст
   - Прочитай страницы подтемы из PDF
   - Создай краткий пересказ → сохрани в полный/[Часть]/Глава [N]/подтемы/[N.M. Название].md
3. После всех подтем главы → создай файл главы с ссылками на подтемы
4. После всех глав части → создай файл части с ссылками на главы
5. Для "по главам/": объедини все подтемы каждой главы в один файл (в 3 раза короче)
6. Для "краткий/": объедини все главы в единое повествование
7. Создай оглавления для полный/ и по главам/
8. Создай корневой файл [Название книги].md с описанием и ссылками
9. Добавь markdown-ссылки рядом с Obsidian-ссылками во всех файлах

**Важно:**
- В файлах "по главам/" подзаголовки должны быть ссылками на подтемы из "полный/"
- В конце каждой главы в "по главам/" добавь ссылку на полную главу
- Используй Obsidian-ссылки [[...]] и markdown-ссылки [...](...)
- Сохраняй всю важную информацию, но в сжатом виде

Начни с чтения оглавления.
```

## Пример заполнения:

```
Мне нужно создать конспект книги по структуре, аналогичной @conspects/data-intensive-applications-summary.

**PDF файл:** conspects/clean-code/original/clean_code.pdf
**Страницы с оглавлением:** 3-8
**Корневая папка:** conspects/clean-code-summary/ru/

[Далее весь текст шаблона выше]
```
