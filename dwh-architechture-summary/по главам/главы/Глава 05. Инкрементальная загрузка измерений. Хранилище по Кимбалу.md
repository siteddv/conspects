# Глава 05. Инкрементальная загрузка измерений. Хранилище по Кимбалу

Пятая глава курса посвящена переходу от полной перезагрузки к инкрементальной загрузке измерений. Рассматриваются методы определения измененных записей, загрузка только новых и измененных данных, оптимизация процесса загрузки.

Инкрементальная загрузка позволяет загружать только изменения вместо полной перезагрузки всех данных, что значительно снижает нагрузку на систему и ускоряет процесс обновления хранилища. Для определения изменений используется механизм отслеживания даты последнего обновления (`last_update`) или сравнение хэшей данных.

Рассматриваются различные подходы к инкрементальной загрузке: загрузка новых записей (отсутствующих в Core), загрузка измененных записей (с измененными атрибутами), обработка удаленных записей (отсутствующих в источнике). Для новых записей создаются новые строки в измерениях. Для измененных записей могут создаваться новые версии с историчностью или обновляться существующие в зависимости от бизнес-логики.

Создаются процедуры для инкрементальной загрузки измерений, которые проверяют наличие записей в Core, сравнивают данные и загружают только изменения. Оптимизируется процесс загрузки через использование индексов и эффективных SQL-запросов.
