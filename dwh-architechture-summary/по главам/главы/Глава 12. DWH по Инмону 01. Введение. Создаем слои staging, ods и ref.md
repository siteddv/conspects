# Глава 12. DWH по Инмону 01. Введение. Создаем слои staging, ods и ref

Двенадцатая глава курса посвящена изучению методологии Инмона (Inmon Data Warehouse) — альтернативного подхода к построению хранилищ данных. Рассматривается введение в методологию Инмона, создание слоев staging, ODS и ref.

Модель Инмона строится в третьей нормальной форме (3NF), в отличие от Кимбалла, который использует денормализованную схему "звезда". Основная идея — разделение данных по слоям: Enterprise Data Warehouse (EDW) как централизованная база в 3NF и Data Marts как специализированные представления для BI.

Подход Инмона включает создание нескольких слоев: staging (временное хранилище сырых данных, аналогично методу Кимбалла), ODS (Operational Data Store — операционное хранилище данных с данными в реальном времени, содержит актуальные данные из операционных систем), ref (справочники и нормализованные данные, содержит справочную информацию в нормализованном виде).

Слой staging создается для временного хранения сырых данных из источников без преобразований. Слой ODS содержит операционные данные в реальном времени, которые могут использоваться для оперативной аналитики. Слой ref содержит справочники и нормализованные данные, которые используются в других слоях хранилища.

Преимущества подхода Инмона: более надежная структура, легче адаптировать к изменениям в источниках, лучше подходит для сложных связей между данными. Недостатки: сложнее в реализации, требует больше времени на разработку, медленнее на агрегатах по сравнению с подходом Кимбалла.
