# Глава 13. DWH по Инмону 02. Создаем слои integration и dds

Тринадцатая глава курса продолжает изучение методологии Инмона. Рассматривается создание слоев integration и dds (Dimensional Data Store).

Слой integration содержит нормализованные данные в 3NF, что обеспечивает гибкость при изменениях в источниках данных. Данные структурированы в соответствии с правилами нормализации, что позволяет легко адаптировать хранилище к изменениям в источниках.

Слой DDS содержит денормализованные данные, оптимизированные для аналитических запросов. Данные организованы в виде схемы "звезда" или "снежинка" для быстрого построения отчетов. DDS создается на основе integration слоя через процедуры трансформации данных.
