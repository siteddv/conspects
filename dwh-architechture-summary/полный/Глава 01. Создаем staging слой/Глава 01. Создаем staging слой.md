## Краткое содержание

Курс начинается с объяснения необходимости хранилищ данных. На примере магазина компакт-дисков показано, что данные часто распределены по разным системам, что затрудняет построение аналитических отчетов. Решение — создание отдельного хранилища данных (DWH), которое объединяет данные из разных источников.

Практическая часть начинается с создания Staging Layer — промежуточного хранилища сырых данных. Создается отдельная база данных для DWH, схема `staging`, настраивается подключение к базе данных источника через PostgreSQL FDW, создаются таблицы и процедуры для загрузки данных.

Staging-слой расширяется добавлением дополнительных таблиц, создаются процедуры для их загрузки и автоматизации процесса. Данные переносятся в Core Layer — основной слой хранилища с нормализованными данными. Выполняется очистка и нормализация данных, создаются ETL-процедуры, оптимизируется производительность.

Завершающий этап — создание Data Marts (витрин данных) для построения конкретных отчетов. Витрины содержат агрегированные данные, готовые для анализа, и подключаются к BI-системам для визуализации.
