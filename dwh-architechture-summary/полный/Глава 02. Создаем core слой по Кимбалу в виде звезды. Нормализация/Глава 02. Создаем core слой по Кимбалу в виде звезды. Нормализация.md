# Глава 02. Создаем core слой по Кимбалу в виде звезды. Нормализация

Вторая глава курса посвящена созданию основного слоя хранилища данных (Core Layer) по методологии Кимбалла в виде звездообразной схемы. Рассматриваются вопросы нормализации данных и структурирования хранилища.

## Краткое содержание

Глава начинается с разбора вопросов участников по прошлому занятию: обновление схемы базы данных, создание новых подключений в PostgreSQL, работа с временными таблицами, влияние типов данных на производительность. Разбирается домашнее задание по заполнению таблицы случайными данными.

Основная часть посвящена изучению основных слоев хранилища данных: Staging (стейджинг), Core (ядро) и Marts (витрины данных). Рассматривается процесс нормализации данных: первая нормальная форма (1NF) — атомарность данных, вторая нормальная форма (2NF) — зависимость от составного ключа целиком, третья нормальная форма (3NF) — устранение транзитивных зависимостей. Нормализация позволяет избежать дублирования информации, обеспечить согласованность данных и повысить гибкость хранения.
