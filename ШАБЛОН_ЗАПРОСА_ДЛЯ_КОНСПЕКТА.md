# Универсальный запрос для создания конспекта книги

## Инструкция

Используй этот шаблон для создания конспекта любой книги. Замени информацию в квадратных скобках на реальные данные.

---

## ЗАПРОС:

Мне нужно создать конспект книги. Вот информация:

**PDF файл:** `[путь_к_pdf_файлу]` (например: `conspects/[название_книги]/original/[файл].pdf`)

**Страницы с оглавлением:** [номер_первой_страницы] - [номер_последней_страницы] (например: 5-15)

**Структура папок для конспекта:**
- Корневая папка: `conspects/[название_книги]-summary/ru/`
- Внутри создай три папки:
  - `полный/` - для детального конспекта с подтемами
  - `по главам/` - для конспекта по главам (без подтем)
  - `краткий/` - для краткого конспекта всей книги

**Структура конспекта:**
Создай конспект по следующей структуре (на основе оглавления из PDF):
- Если в книге есть части: `Часть X. [Название]/` → `Глава Y. [Название]/` → `подтемы/[Y.Z. Название подтемы].md`
- Если частей нет: `Глава Y. [Название]/` → `подтемы/[Y.Z. Название подтемы].md`

**Процесс создания конспекта:**

1. **Создай файл оглавления:** 
   - Прочитай страницы оглавления из PDF
   - Сохрани структуру в файл `table_of_contents.txt` в корне проекта

2. **Создай полный конспект (с подтемами):**
   - В цикле для каждой подтемы:
     - Очисти контекст (для каждой новой подтемы)
     - Прочитай соответствующие страницы из PDF (используй `read_pdf_pages.py` или аналогичный инструмент)
     - Создай краткий пересказ подтемы (сохрани всю важную информацию, но в сжатом виде)
     - Сохрани в файл: `полный/[Часть X]/Глава Y/[подтемы/Y.Z. Название].md`
   - После обработки всех подтем главы:
     - Прочитай все файлы подтем главы
     - Создай краткий пересказ главы с ссылками на подтемы
     - Сохрани в: `полный/[Часть X]/Глава Y/Глава Y. [Название].md`
   - После обработки всех глав части:
     - Прочитай все файлы глав части
     - Создай краткий пересказ части с ссылками на главы
     - Сохрани в: `полный/[Часть X]/Часть X. [Название].md`

3. **Создай конспект по главам:**
   - Для каждой главы:
     - Прочитай все файлы подтем этой главы из `полный/`
     - Создай конспект главы, объединяя информацию из всех подтем (объем должен быть примерно в 3 раза меньше, чем сумма подтем)
     - Сохрани в: `по главам/[Часть X]/главы/Глава Y. [Название].md`
     - В файле главы:
       - Каждый подзаголовок (например, `## Y.Z. Название подтемы`) должен быть ссылкой на соответствующую подтему в `полный/`: `## [[Y.Z. Название подтемы]]`
       - В конце файла добавь ссылку на полную главу: `**Подробнее:** [[Глава Y. [Название]]]`
   - После обработки всех глав части:
     - Создай файл части с ссылками на главы
     - Сохрани в: `по главам/[Часть X]/Часть X. [Название].md`

4. **Создай краткий конспект:**
   - Прочитай все файлы глав из `по главам/`
   - Создай единый конспект всей книги в одном файле
   - Объем: 25000-30000 символов
   - НЕ упоминай явно главы и части в тексте, сделай единое повествование
   - Сохрани в: `краткий/Конспект книги.md`

5. **Создай оглавления:**
   - `полный/Оглавление.md` - со ссылками на все части, главы и подтемы
   - `по главам/Оглавление.md` - только со ссылками на части и главы (без подтем)

6. **Создай корневой файл книги:**
   - В корне папки `conspects/[название_книги]-summary/` создай файл `[Название книги].md`
   - Содержимое:
     - Заголовок с названием книги
     - Автор
     - Оригинальное название (если есть)
     - Краткое описание книги
     - Ссылки на три уровня конспектов:
       - `[[ru/полный/Оглавление|Полный конспект]]`
       - `[[ru/по главам/Оглавление|Конспект по главам]]`
       - `[[ru/краткий/Конспект книги|Краткий конспект]]`

7. **Добавь markdown-ссылки:**
   - Во всех файлах рядом с каждой Obsidian-ссылкой `[[...]]` добавь обычную markdown-ссылку `[текст](относительный_путь.md)`
   - Относительные пути должны вычисляться от текущего файла

**Важные требования:**
- Используй Obsidian-ссылки в формате `[[название]]` или `[[путь|текст]]`
- Все файлы в формате Markdown (.md)
- Сохраняй всю важную информацию, но в сжатом виде
- Структура папок должна точно соответствовать структуре книги
- В файлах глав в `по главам/` объем должен быть примерно в 3 раза меньше, чем сумма подтем

**Начни с чтения оглавления и создания структуры папок. Если есть вопросы по реализации, задай их перед началом работы.**
